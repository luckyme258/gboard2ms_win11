## 微软拼音 .dat 文件格式规则详解

基于对 `UserDefinedPhrase.dat` 的成功逆向分析，我总结出以下完整规则：

---

### 一、文件整体结构

```
┌─────────────────────────────────────────────────────────────┐
│                     文件头 (48 字节)                         │
├─────────────────────────────────────────────────────────────┤
│                     偏移表 (4 × N 字节)                      │
├─────────────────────────────────────────────────────────────┤
│                     词条区 (变长)                            │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 词条 0 (变长)                                        │   │
│  ├─────────────────────────────────────────────────────┤   │
│  │ 词条 1 (变长)                                        │   │
│  ├─────────────────────────────────────────────────────┤   │
│  │ ...                                                 │   │
│  └─────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘
```

---

### 二、文件头 (48 字节)

| 偏移 | 长度 | 字段 | 说明 | 示例值 |
|------|------|------|------|--------|
| 0 | 8 | Magic | 文件标识 | `mschxudp` |
| 8 | 8 | Version | 版本号 | `02 00 60 00 01 00 00 00` |
| 16 | 4 | OffsetTableStart | 偏移表起始位置 | 64 (0x40) |
| 20 | 4 | EntryAreaStart | 词条区起始位置 | 72 (0x48) |
| 24 | 4 | TotalLength | 文件总长度 | 146 |
| 28 | 4 | EntryCount | 词条总数 | 2 |
| 32 | 4 | ExportTimestamp | 导出时间戳 | |
| 36 | 12 | Padding | 填充 | 全 0 |

---

### 三、偏移表

每个词条 4 字节，存储该词条在词条区中的偏移量（从词条区起始位置计算）。

```
偏移表 = [offset_0, offset_1, offset_2, ...]
```

---

### 四、词条结构

每个词条由三部分组成：**头部(16字节) + 编码字段 + 短语字段**

#### 4.1 头部 (16 字节)

| 偏移 | 长度 | 字段 | 说明 | 示例值 |
|------|------|------|------|--------|
| 0 | 4 | Marker | 固定标识 | `10 00 10 00` (0x00100010) |
| 4 | 2 | CodeTotalLen | 编码字段总长度 + 12 | 22 (0x0016) |
| 6 | 1 | Position | 候选位置 (1-9) | 1 |
| 7 | 1 | UnknownFlag | 未知标志，固定 0x06 | 6 |
| 8 | 4 | Reserved | 保留字段，全 0 | 0 |
| 12 | 4 | Timestamp | 词条时间戳 | |

**公式：** `编码字段长度 = CodeTotalLen - 12`

#### 4.2 编码字段 (变长)

结构：`[编码] + [\0\0] + [短语前半部分]`

| 部分 | 编码 | 说明 |
|------|------|------|
| 编码 | UTF-16LE | 用户输入的快捷编码，如 `ky` |
| 分隔符 | `00 00` | 两个空字节，作为分隔 |
| 短语前半部分 | UTF-16LE | 完整短语的前 2 个字符 |

#### 4.3 短语字段 (变长)

结构：`[短语后半部分] + [\0\0]`

| 部分 | 编码 | 说明 |
|------|------|------|
| 短语后半部分 | UTF-16LE | 完整短语的剩余字符 |
| 结束符 | `00 00` | 两个空字节，标记结束 |

---

### 五、短语拆分规则

**核心规则：完整短语 = 编码字段(分隔符之后) + 短语字段(结束符之前)**

示例：短语 `可以的啊啊` (5 个字符)

```
编码字段: ky + \0\0 + 可以   → 分隔符后是 "可以"
短语字段: 的啊啊 + \0\0      → 结束符前是 "的啊啊"
完整短语: "可以" + "的啊啊" = "可以的啊啊"
```

**拆分规律：**
- 当短语长度 ≥ 2 时：前 2 个字符放入编码字段，剩余字符放入短语字段
- 当短语长度 < 2 时：全部放入编码字段，短语字段为空

| 短语 | 长度 | 编码字段存放 | 短语字段存放 |
|------|------|-------------|-------------|
| `可以的啊啊` | 5 | `可以` (2字) | `的啊啊` (3字) |
| `假面骑士zzz` | 7 | `假面` (2字) | `骑士zzz` (5字) |
| `M哥` | 2 | `M哥` (2字) | `` (0字) |
| `a` | 1 | `a` (1字) | `` (0字) |

---

### 六、完整示例：`ky` → `可以的啊啊`

**原始数据：**
```
编码字段原始: 6b 00 79 00 00 00 ef 53 e5 4e
短语字段原始: 84 76 4a 55 4a 55 00 00
```

**解析过程：**

```
编码字段:
  6b 00     → 'k'
  79 00     → 'y'
  00 00     → 分隔符 \0\0
  ef 53     → '可'
  e5 4e     → '以'
  编码字段中的短语部分 = "可以"

短语字段:
  84 76     → '的'
  4a 55     → '啊'
  4a 55     → '啊'
  00 00     → 结束符
  短语字段中的短语部分 = "的啊啊"

完整短语 = "可以" + "的啊啊" = "可以的啊啊"
```

---

### 七、关键公式总结

```
编码字段实际存储内容 = 编码(UTF-16LE) + b'\x00\x00' + 短语前2字符(UTF-16LE)
短语字段实际存储内容 = 短语剩余字符(UTF-16LE) + b'\x00\x00'

CodeTotalLen = len(编码字段) + 12

完整短语 = 编码字段[分隔符后] + 短语字段[结束符前]
```

### 八、代码实现要点

```python
# 拆分短语
if len(phrase) >= 2:
    phrase_part1 = phrase[:2]   # 前2字
    phrase_part2 = phrase[2:]   # 剩余
else:
    phrase_part1 = phrase
    phrase_part2 = ""

# 构建字段
code_field = code.encode('utf-16le') + b'\x00\x00' + phrase_part1.encode('utf-16le')
phrase_field = phrase_part2.encode('utf-16le') + b'\x00\x00'

# 头部
code_total_len = len(code_field) + 12
header = struct.pack('<I', 0x00100010)      # marker
header += struct.pack('<H', code_total_len) # code_total_len
header += struct.pack('<B', position)      # 候选位置
header += b'\x06'                           # unknown_flag
header += struct.pack('<I', 0x00000000)    # reserved
header += struct.pack('<I', timestamp)     # timestamp
```

这就是微软拼音 `.dat` 文件的完整格式规则！
